# Expense Tracker Discord Bot

โยนรูปสลิปโอนเงิน/ใบเสร็จให้บอทใน Discord (ทัก DM หรือส่งในแชนแนล) แล้วบอทจะ OCR อ่านรูป
แยกยอดเงิน/วันที่/เวลา/ร้านค้า/หมวดหมู่ให้อัตโนมัติ **บันทึกทันที** แล้วตอบกลับเป็นสรุป
ที่มีปุ่ม **แก้ไข / สลับรายรับ-รายจ่าย / ลบ** ให้แก้ไขได้ทันทีถ้าอ่านผิด

ถ้า OCR อ่านยอดเงินไม่ออก บอทจะยังบันทึกรายการไว้ (ไม่หายไปไหน) แต่จะไม่นับยอดนั้นในสรุป
จนกว่าจะกดแก้ไขกรอกยอดเอง — กันไม่ให้ยอดผิดหลุดเข้าไปในสรุปเงินแบบเงียบๆ

## ต้องมีอะไรก่อน

- Node.js 18 ขึ้นไป (ตรวจด้วย `node -v`)
- VPS หรือเครื่องที่เปิดค้างไว้ได้ตลอดเวลา (ให้บอทออนไลน์)
- บัญชี Discord

## ขั้นตอนที่ 1: สร้าง Discord Bot

1. ไปที่ https://discord.com/developers/applications แล้วกด **New Application** ตั้งชื่อตามใจ
2. ไปแท็บ **Bot** (ถ้ายังไม่มี bot user ให้กด Add Bot)
3. ที่ **Privileged Gateway Intents** ให้เปิด **MESSAGE CONTENT INTENT** (จำเป็นมาก ถ้าไม่เปิดบอทจะอ่านรูปที่แนบมาไม่ได้)
4. กด **Reset Token** เพื่อดู token แล้วคัดลอกเก็บไว้ (จะใช้เป็น `DISCORD_TOKEN`)
5. ไปแท็บ **General Information** คัดลอก **Application ID** เก็บไว้ (จะใช้เป็น `CLIENT_ID`)
6. ไปแท็บ **OAuth2 > URL Generator**
   - Scopes: เลือก `bot`
   - Bot Permissions: เลือก `Send Messages`, `Read Message History`, `Attach Files`, `Use Slash Commands`
   - คัดลอกลิงก์ที่ได้ เปิดในเบราว์เซอร์ แล้วเลือก server ที่จะเชิญบอทเข้าไป (ต้องมี server อย่างน้อย 1 อัน แม้จะใช้แบบ DM ก็ตาม เพราะ Discord กำหนดให้บอทกับผู้ใช้ต้องอยู่ server เดียวกันก่อนถึงจะทัก DM กันได้)

## ขั้นตอนที่ 2: หา User ID ของตัวเอง (สำหรับ OWNER_ID)

1. ใน Discord app: Settings > Advanced > เปิด **Developer Mode**
2. คลิกขวาที่ชื่อตัวเอง (ที่ไหนก็ได้) > **Copy User ID**
3. เก็บไว้เป็นค่า `OWNER_ID` — บอทจะประมวลผลรูปเฉพาะจากไอดีนี้เท่านั้น คนอื่นทักไปก็จะเงียบ ไม่บันทึกอะไร (กันข้อมูลการเงินรั่วไปคนอื่น)

## ขั้นตอนที่ 2.5: (แนะนำ) สร้างแชนแนลสำรองข้อมูล

ทุก 3 วัน (เที่ยงคืน) บอทจะส่งไฟล์ `records.json` + `merchant_memory.json` เป็นไฟล์แนบเข้าแชนแนลนี้
อัตโนมัติ กันข้อมูลหายถ้า VPS พัง:

1. สร้าง text channel ใหม่ใน server ที่เชิญบอทเข้าไป ตั้งเป็น **private** เห็นแค่ตัวเองกับบอท
2. คลิกขวาที่ channel > **Copy Channel ID** (ต้องเปิด Developer Mode แล้วเหมือนขั้นตอนที่ 2)
3. เก็บไว้เป็นค่า `BACKUP_CHANNEL_ID`

ถ้าไม่ตั้งค่านี้ บอทจะไม่สำรองข้อมูลอัตโนมัติ แต่ยังใช้คำสั่ง `/backup_now` เองได้ทุกเมื่อ

## ขั้นตอนที่ 3: ติดตั้งบน VPS

```bash
# อัปโหลดโฟลเดอร์ discord-bot ทั้งโฟลเดอร์ขึ้น VPS (scp, git, หรือ sftp ก็ได้) แล้ว
cd discord-bot
npm install

cp .env.example .env
nano .env   # กรอก DISCORD_TOKEN, CLIENT_ID, GUILD_ID (แนะนำ), OWNER_ID, BACKUP_CHANNEL_ID ตามที่เก็บไว้ในขั้นตอนที่แล้ว
```

`GUILD_ID` คือ Server ID ของ server ที่เชิญบอทเข้าไป (คลิกขวาที่ไอคอน server > Copy Server ID
ต้องเปิด Developer Mode ก่อนเหมือนกัน) — ใส่ไว้จะได้ slash command ใช้งานได้ทันที ถ้าไม่ใส่
ต้องรอนานสุดถึง 1 ชั่วโมงกว่าคำสั่งจะขึ้นทุก server

`BACKUP_CRON` ปรับเวลาสำรองข้อมูลได้ (ค่าเริ่มต้น `0 0 */3 * *` = เที่ยงคืนทุก 3 วัน ตามเวลาเครื่อง VPS)

## ขั้นตอนที่ 4: ลงทะเบียน slash command แล้วรันบอท

```bash
npm run deploy-commands   # ทำครั้งเดียว (หรือทุกครั้งที่แก้คำสั่งใน deploy-commands.js)
npm start                 # ทดสอบรันดูก่อนว่าออนไลน์ปกติ (Ctrl+C เพื่อหยุด)
```

ถ้าเห็น `✅ บอทออนไลน์แล้ว: ชื่อบอท#1234` แปลว่าใช้ได้แล้ว

### ให้บอทรันค้างไว้ตลอด (แนะนำ pm2)

```bash
npm install -g pm2
pm2 start bot.js --name expense-bot
pm2 save
pm2 startup    # จะโชว์คำสั่งให้ copy ไปรันอีกที เพื่อให้ pm2 auto-start ตอนเครื่อง reboot
```

ดูล็อกด้วย `pm2 logs expense-bot`, รีสตาร์ทด้วย `pm2 restart expense-bot`

## วิธีใช้งาน

- **บันทึกอัตโนมัติ**: ส่งรูปสลิป/ใบเสร็จหาบอท (DM หรือแชนแนลที่บอทอยู่) — บอทจะตอบกลับข้อความ
  "🔎 กำลังอ่านสลิป..." แล้วอัปเดตเป็นสรุปรายการพร้อมปุ่มภายในไม่กี่วินาที
- **แก้ไขรายการ**: กดปุ่ม ✏️ แก้ไข ใต้ข้อความสรุป จะเด้งฟอร์มให้แก้ยอดเงิน/วันที่/ร้าน/หมวดหมู่/บันทึกได้
  แก้หมวดหมู่ทีไหน บอทจะจำไว้ (ดูหัวข้อ "จำหมวดหมู่ตามร้าน" ด้านล่าง)
- **สลับรายรับ-รายจ่าย**: กดปุ่ม 🔄 (ค่าเริ่มต้นเดาเป็นรายจ่ายเสมอ)
- **ลบรายการ**: กดปุ่ม 🗑️
- **`/summary [month]`**: สรุปยอดรายรับ-รายจ่ายของเดือน (ไม่ใส่ = เดือนนี้) เช่น `/summary month:2026-07`
- **`/list [month] [count]`**: แสดงรายการล่าสุด
- **`/export [format]`**: ส่งออกรายการทั้งหมดเป็นไฟล์ **Excel (.xlsx)** (ค่าเริ่มต้น — มี 2 ชีต:
  "รายการทั้งหมด" กับ "สรุปรายเดือน" จัดฟอร์แมตสี/ตัวเลขเงินให้แล้ว) หรือเลือก `format: CSV`
  ก็ได้ถ้าอยากได้ไฟล์ดิบธรรมดา
- **`/delete_last`**: ลบรายการล่าสุด เผื่อบอทจำผิดแล้วอยากลบทิ้งไว
- **`/add`**: เพิ่มรายการด้วยมือแบบเร็ว ไม่ต้องมีรูปสลิป (เช่น จ่ายเงินสด) เช่น
  `/add amount:65 merchant:ร้านข้าวแกง type:รายจ่าย` — ไม่ใส่หมวดหมู่ก็ได้ บอทจะเดาให้
  (หรือใช้ค่าที่เคยจำไว้จากร้านเดิม ถ้ามี)
- **`/backup_now`**: สั่งสำรองข้อมูลเข้าแชนแนลสำรองทันที ไม่ต้องรอถึงเที่ยงคืน (ต้องตั้ง
  `BACKUP_CHANNEL_ID` ไว้ก่อน)

### กันบันทึกซ้ำ

ถ้ารายการใหม่ (จากรูปสลิปหรือ `/add`) มี วันที่ + ยอดเงิน + ชื่อร้าน ตรงกับรายการที่มีอยู่แล้ว
บอทจะยังบันทึกให้ตามปกติ แต่ขึ้นหัวข้อเตือน "⚠️ ดูเหมือนจะซ้ำกับรายการที่มีอยู่" ให้เช็คเอง
ว่าไม่ได้ส่งสลิปใบเดิมซ้ำ (เผื่อร้าน/ยอด/วันที่ตรงกันโดยบังเอิญจริงๆ ก็ปล่อยไว้ได้ ไม่ได้ลบให้อัตโนมัติ)

### จำหมวดหมู่ตามร้าน

ทุกครั้งที่แก้หมวดหมู่ผ่านปุ่ม ✏️ แก้ไข (หรือใส่ตรงๆ ผ่าน `/add`) บอทจะจำคู่ "ชื่อร้าน → หมวดหมู่"
ไว้ ครั้งต่อไปถ้าสลิปจากร้านเดิม (ชื่อตรงกัน ไม่สนตัวพิมพ์เล็ก-ใหญ่/ช่องว่างหน้า-หลัง) เข้ามาอีก
บอทจะใส่หมวดหมู่ที่เคยแก้ไว้ให้อัตโนมัติแทนการเดาใหม่ทุกครั้ง

## ข้อมูลเก็บที่ไหน

รายการทั้งหมดอยู่ในไฟล์ `data/records.json` และหมวดหมู่ที่จำไว้ตามร้านอยู่ใน
`data/merchant_memory.json` (สร้างอัตโนมัติตอนรันครั้งแรก) เป็น JSON ธรรมดา เปิดอ่านตรงๆ ได้
ถ้าตั้งค่า `BACKUP_CHANNEL_ID` ไว้ บอทจะส่งไฟล์ทั้งสองนี้เข้าแชนแนลสำรองให้อัตโนมัติทุก 3 วัน
อยู่แล้ว (หรือสั่งด้วย `/backup_now` เมื่อไหร่ก็ได้)

## หมายเหตุเรื่องความแม่นยำของ OCR

บอทใช้ระบบ parsing เดียวกับเว็บแอพ `expense-tracker.html` (grayscale + ปรับคอนทราสต์ก่อนอ่าน,
แก้ปัญหา Unicode สระอำที่ Tesseract มักอ่านแยกส่วน) แต่ OCR ก็ยังมีโอกาสอ่านผิดได้อยู่ดี
โดยเฉพาะยอดเงินที่เป็นตัวหนา/สีต่างจากพื้นหลังมาก ถ้าอ่านไม่ได้บอทจะบันทึกรายการไว้แต่ขึ้นเตือน
⚠️ ให้กดแก้ไขกรอกเอง แทนที่จะเดาผิดแล้วเงียบไป

## ความปลอดภัย

- ห้าม commit ไฟล์ `.env` ขึ้น git (มี token อยู่ในนั้น) — ถ้าใช้ git ให้เพิ่ม `.env` และ `data/`
  ลงใน `.gitignore`
- ตั้งค่า `OWNER_ID` เสมอ ไม่งั้นใครก็ตามที่ทักบอทหรือเชิญเข้า server จะสั่งบันทึก/ลบ/ดูข้อมูล
  การเงินของนายได้
